Explaining Ruby Classes

A big part of this week's challenges was about classes in Ruby. The nicest analogy I found describing classes was "A class is like a blueprint of a house. Once you've made is you can use it over and over again". Essentially we make classes for code we have to use over and over again and since we are lazy and want to keep our code DRY (don't repeat yourself) we use classes. In this post we'll be building a small tool to explain the concept of classes.

Defining a class is very simple! We'll call our class FamilyMembers. By convention you don't seperate the class name with an underscore like we do for methods but like this: 

class FamilyMembers

Very easy right? Let's add some functionality. Our tool is simply going to tell us how old certain family member is and what his/her name is. It's a useless tool but should be a good example. 

class FamilyMemebers

			def initialize(name, age)
			@name = name
			@age = age
			end

			def name
			@name
			end

			def set_name(name)
			@name = name
			end

			def age
			@age
			end

			def set_age(age)
			@age = age
			end

			end

This is going to look a tiny bit confusing at first. I'll explain everything now starting at the top. The initialize method I defined first is a special method in Ruby that will be run automatically when a class is called. As soon as FamilyMembers get called the initialize method is executed. Well the next question is what this method is actually doing. It's quite simple actually. The initialize method creates "instance variables" and like the name suggests they are variables used in instances of a class. The "@" sign assigns instance variables and they can be accessed throughout and entire class. What we're doing in the first method is creating a variable that can be used in the entire class, at least if it's ran as in instance but we'll get to that later.

The other four methods are quite simple as they either return a variable or assign a new value to one of the instance variables. 

class FamilyMemebers

			def name
			@name # returns @name variable value
			end

			def set_name(name)
			@name = name # assigns new value to @name when called with argument
			end

			end

Now this seems boring and useless but now we shall start using the class magic. Remember I was talking about instances of a class? That exactly what we'll be doing now! When we want to call an instance of a method we do it like this: 

new_instance = FamiliyMembers.new("Hans", 37)
			# this creates a new instance of the the FamiliyMembers class with the arguments 
			# passed ("Hans", 37)

Now we can call the methods defined in the class like this: 

new_instance.name
			# Will return the value assigned to @name

			=> "Hans"

			new_instance.age

			=> 37

Changing a value now works just the same way! I bet you could do it yourself now! 

new_instance.set_name("Frank")

			new_instance.name

			=> "Frank"

So that should be all for today! We could assign new names and ages all day long if we wanted. Using a class we minimized the amount of code we have which is always a great thing! Hope it all makes sense to you!
