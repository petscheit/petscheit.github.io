Explaining Ruby Classes

A big part of this week's challenges was about classes in Ruby. The nicest analogy I found describing classes was "A class is like a blueprint of a house. Once you've made is you can use it over and over again". Essentially we make classes for code we have to use over and over again and since we are lazy and want to keep our code DRY (don't repeat yourself) we use classes. In this post we'll be building a small tool to explain the concept of classes.

This is going to look a tiny bit confusing at first. I'll explain everything now starting at the top. The initialize method I defined first is a special method in Ruby that will be run automatically when a class is called. As soon as FamilyMembers get called the initialize method is executed. Well the next question is what this method is actually doing. It's quite simple actually. The initialize method creates "instance variables" and like the name suggests they are variables used in instances of a class. The "@" sign assigns instance variables and they can be accessed throughout and entire class. What we're doing in the first method is creating a variable that can be used in the entire class, at least if it's ran as in instance but we'll get to that later.

The other four methods are quite simple as they either return a variable or assign a new value to one of the instance variables.
