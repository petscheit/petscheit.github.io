SQL Injection

SQQL injections are a very common kind of cyber attacks. Their aim is to gain full access into a websites/companies database. Once a hacker has full access to a database he can do what ever he wants, copy files, modify dataa and even delete the entire thing. Most big data thefts we hear about in the news quite frequently are SQL injections. So what exactly is a SQL injection and how do they work?

First we shall take a look at how SQL works and what it is. SQL is the language we use to communitcate with a database (a database containsone or more tables). The tables work just like an excel table. Basically every website uses a database of some sort to store data like login information. Let's take a look at how SQL looks. When you type your username and password the website treats your entries as a string -> nothing can really happen. But now imagine you're able to pass actual code through the login screen that is interpreted by SQL and then executed. Essentially that's how an SQL injection works.

So how does that work? How can we trick a website into interpreting our input not as a string? Well there are a lot of different approaches but the standart procedure is to test how a database handles "bad inputs". One can test that by adding a quotes into the username (e.g. 'name@email.com''). Playing around will show how the database works and where it might have weaknesses. Once we've figured out how we can pass code that is not recogized as a string anymore the doors are basically open. It's quite scary how simple this is in theory and how often it happens to huge corperations
